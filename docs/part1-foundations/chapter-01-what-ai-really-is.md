# Chapter 1: What AI Really Is

AI is a broad term covering systems that perform tasks normally requiring human judgment. Most of what organizations encounter today falls into a few well-defined categories.

---

## AI vs Automation vs Analytics

| Type | What It Does | Example |
|---|---|---|
| **Automation** | Executes fixed rules | Invoice routing, form processing |
| **Analytics** | Finds patterns in data | Sales dashboards, churn prediction |
| **AI / ML** | Learns from examples, handles variability | Fraud detection, document classification |
| **Generative AI** | Creates new content from prompts | Drafting, summarization, code generation |

These are not a hierarchy. Many useful systems are purely automated. Not every problem needs AI.

---

## Generative AI Explained Simply

Large language models (LLMs) are trained on vast text datasets to predict the next token in a sequence. The result is a system that can write, summarize, translate, answer questions, and generate code — but that has no understanding, memory, or ground truth. It produces plausible text, not verified text.

---

## Machine Learning Basics

Most AI systems learn from labeled examples rather than explicit rules. Key concepts:

- **Training data** — examples the model learns from
- **Model** — the learned function that maps input to output
- **Inference** — running the trained model on new inputs
- **Fine-tuning** — adapting a pre-trained model for a specific task

---

## Common Myths

| Myth | Reality |
|---|---|
| AI understands your domain | It predicts tokens; domain accuracy depends on training data |
| AI is always improving | Benchmarks improve; production reliability improves more slowly |
| AI will replace entire roles | Tasks shift; wholesale role elimination is rare in practice |
| More data always helps | Data quality matters more than volume for most applications |

---

## How Organizations Use AI Today

- **Productivity tools** — writing assistance, meeting summaries, code completion
- **Customer-facing** — chatbots, search, recommendation
- **Operations** — document processing, classification, anomaly detection
- **Decision support** — risk scoring, forecasting, analytics augmentation

---

## Key Takeaways

- AI, automation, and analytics are distinct — match the tool to the problem.
- Generative AI produces plausible output, not guaranteed-correct output.
- Most organizational AI use cases are narrow, not general.

**Common mistake:** Evaluating AI from vendor demos. Demos are curated; production is messy.
