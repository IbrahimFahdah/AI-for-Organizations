# Chapter 11: Identifying and Prioritizing AI Use Cases

The most important AI decision most organizations make is not which model to use — it is which problem to solve. Use case selection determines whether AI investment creates value or waste.

---

## Opportunity Assessment

A good AI use case has:

- **Volume** — high frequency makes automation worthwhile
- **Repetition** — consistent enough that a model can generalize
- **Evaluability** — clear criteria for what "correct" looks like
- **Data availability** — training examples, documents, or structured data exist
- **Tolerable error rate** — errors are detectable and correctable

A poor AI use case: one-off decisions, highly novel situations, tasks requiring deep contextual judgment, or cases where errors are catastrophic and invisible.

---

## ROI vs Complexity Analysis

Plot candidate use cases on two dimensions:

```
High ROI │  [Quick wins]     [Strategic bets]
         │
         │
Low ROI  │  [Skip]           [Questionable]
         └──────────────────────────────────
              Low complexity    High complexity
```

**Quick wins** — high ROI, low complexity. Execute first.
**Strategic bets** — high ROI, high complexity. Require dedicated investment.
**Skip** — low ROI regardless of complexity.

## Prioritization Frameworks

Simple scoring model (1–5 scale per criterion):

| Criterion | Weight | Use Case A | Use Case B |
|---|---|---|---|
| Strategic alignment | 25% | | |
| Business value (cost/revenue impact) | 30% | | |
| Feasibility (data, talent, timeline) | 25% | | |
| Risk level (lower = better score) | 20% | | |
| **Weighted total** | 100% | | |

## AI Portfolio Management

Treat AI use cases as a portfolio:

- **Run** — live in production, being monitored and optimized
- **Build** — in active development
- **Explore** — in assessment or pilot
- **Retire** — deprecated or replaced

Review the portfolio quarterly. Kill underperforming projects; reinvest in successful ones.

---

## Quick Wins vs Strategic Investments

Quick wins (weeks to months) build organizational confidence, produce tangible results, and fund the case for larger investments. They should be:
- Narrow in scope
- Easy to measure
- Reversible if they fail

Strategic investments (months to years) require different governance: dedicated funding, executive sponsorship, and longer measurement timelines.

---

## Key Takeaways

- Use case selection is the highest-leverage AI decision most organizations make.
- Score candidates consistently using a shared framework to avoid selection by enthusiasm.
- Maintain a portfolio view; kill underperformers rather than sustaining them indefinitely.

**Common mistake:** Selecting use cases based on what is technically interesting rather than what is strategically valuable.
